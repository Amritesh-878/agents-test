name: status-update
description: Write a project status/progress/report update. Use when asked to report status, update progress, summarize project state, write a status report, or prep an update for peers/standup.
---

Update the EXISTING doc — default `PROGRESS.md`. Do not create new markdown files unless explicitly asked.

Rules:
- Be direct. Facts only. No storytelling, no narrative arc, no preamble, no "the journey".
- Terse bullets under plain headings: Done / In flight / Open risks / Decisions needed / Next.
- State what works, what's broken, what's blocked, what you need — nothing else.
- No adjectives selling the work. No "we successfully…", "exciting", "robust", "solid journey".
- Numbers and specifics over prose (test count, real metrics, exact tradeoffs).
- Append a dated `## Status — YYYY-MM-DD` section if one doesn't exist; otherwise update it.
- If the user names a different existing doc, update that one instead.
